# 1. Идея проекта

Платформа предназначена для создания пользовательских учебных наборов, основанных на изображениях и связанных с ними структурированными данными.

Пользователь самостоятельно создает набор изображений, задает необходимые поля описания объектов и заполняет их значениями. После этого система автоматически предоставляет различные режимы обучения и проверки знаний.

Основная идея проекта заключается в разделении данных и способов их изучения.

Каждый набор представляет собой единую базу знаний, поверх которой работают различные режимы обучения.

Таким образом, одна и та же информация может использоваться для карточек, тестов, экзаменов, интервального повторения и других форм обучения без необходимости создавать отдельные материалы.

# 2. Целевая аудитория

Основной аудиторией являются пользователи, которым необходимо изучать большое количество визуальной информации.

Возможные категории пользователей:

- студенты медицинских специальностей;
- студенты художественных направлений;
- преподаватели;
- авторы образовательных курсов;
- школьники;
- любители искусства;
- биологи;
- фотографы;
- дизайнеры;
- все пользователи, изучающие объекты по изображениям.

Проект не ограничивается конкретной предметной областью.

# 3. Основные сущности

## Пользователь

Каждый пользователь имеет собственный аккаунт.

Пользователь может:

- создавать наборы;
- редактировать наборы;
- проходить обучение;
- просматривать статистику;
- делиться своими наборами.

В дальнейшем количество создаваемых наборов может ограничиваться тарифным планом.

## Набор

Набор является основной сущностью системы.

Набор представляет собой коллекцию объектов одной тематики.

Например:

- Русская живопись
- Анатомия человека
- Породы собак
- Архитектурные стили
- Птицы Европы
- Автомобили Formula 1

Каждый пользователь может иметь несколько наборов.

## Объект

Каждый объект состоит из:

- изображения;
- произвольного набора полей.

Примеры:

### Живопись

Изображение

Поля:

- название;
- автор;
- год;
- стиль;
- направление;
- страна.

### Биология

Изображение

Поля:

- название;
- семейство;
- класс;
- среда обитания;
- латинское название.

### Анатомия

Изображение

Поля:

- название структуры;
- система организма;
- латинское название;
- функция.

Система не ограничивает пользователя фиксированным набором атрибутов.

Любые поля создаются пользователем самостоятельно.

# 4. Видимость наборов

Каждый набор может быть:

## Приватным

Доступен только владельцу.

Используется для личного обучения.

## Публичным

Доступ осуществляется по ссылке.

Владелец самостоятельно определяет:

- кто имеет доступ;
- какие режимы обучения доступны;
- могут ли пользователи видеть правильные ответы;
- доступна ли статистика;
- требуется ли авторизация.

Например:

Набор доступен по ссылке.

Разрешены:

- экзамен;
- тест.

Запрещены:

- карточки;
- просмотр базы знаний.

Такой режим позволяет использовать платформу как систему проведения экзаменов.

# 5. Способы наполнения наборов

## Ручное заполнение

Пользователь:

1. создает набор;
2. загружает изображения;
3. создает поля;
4. заполняет значения.

Подходит для небольших коллекций.

## Массовый импорт

Пользователь загружает:

- CSV;
- Excel.

После этого загружает архив изображений.

Система автоматически связывает изображения с данными по имени файла.

## Смешанный режим

Часть данных импортируется автоматически.

Недостающие значения пользователь заполняет вручную.

# 6. Режимы обучения

Все режимы работают поверх одного и того же набора.

## Карточки

Показывается изображение.

Пользователь пытается вспомнить ответ.

После нажатия отображаются выбранные поля.

## Тест

Показывается изображение.

Система генерирует несколько вариантов ответа.

Пользователь выбирает правильный.

## Экзамен

Формируется случайная серия вопросов.

В конце отображается результат.

## Угадай по фрагменту

Показывается часть изображения.

Необходимо определить объект.

## Постепенное раскрытие

Изображение открывается постепенно.

Чем раньше дан правильный ответ, тем выше результат.

## Угадай поле

Система может спрашивать любое поле.

Например:

- Кто автор?
- Какой стиль?
- Как называется объект?
- Какой год?
- Какая система организма?

## Смешанный экзамен

Экзамен автоматически комбинирует различные типы вопросов.

## Свободный ввод

Пользователь самостоятельно вводит ответ без вариантов выбора.

Подходит для более сложного уровня проверки знаний.

## Сравнение объектов

Одновременно отображаются два или несколько похожих объектов.

Необходимо определить различия или выбрать объект, соответствующий заданному условию.

Например:

- Какая картина написана раньше?
- Какой художник относится к импрессионизму?
- Какая птица является хищной?

# 7. Интервальное повторение

Для каждого пользователя ведется история обучения.

Система определяет уровень знания каждого объекта.

Объекты, вызывающие затруднения, появляются чаще.

Хорошо изученные объекты повторяются значительно реже.

Алгоритм повторений работает независимо для каждого пользователя.

# 8. Статистика

## Личная статистика

Для каждого пользователя отображается:

- общий прогресс;
- количество изученных объектов;
- процент правильных ответов;
- история обучения;
- сложные темы;
- наиболее сложные объекты;
- скорость ответов;
- динамика прогресса.

## Статистика набора

Владелец публичного набора может видеть:

- количество прохождений;
- средний результат;
- наиболее сложные вопросы;
- наиболее сложные объекты;
- рейтинг пользователей (при авторизации);
- распределение результатов.

# 9. Архитектура

Основная сущность проекта:

```
Пользователь
      │
      ▼
    Набор
      │
      ▼
   Объект
      │
      ▼
Изображение + Поля
```

Все режимы обучения работают исключительно поверх этой структуры.

Добавление новых режимов не требует изменения модели данных.

# 10. AI (будущие версии)

AI не используется как основной источник данных.

Пользователь самостоятельно создает качественную базу знаний.

## Не использовать AI для

- определения автора картины;
- определения периода;
- автоматического заполнения базы по изображению;
- автоматической генерации содержимого набора без проверки пользователем.

Причины:

- возможные ошибки модели;
- высокая стоимость обработки изображений;
- невозможность гарантировать достоверность информации.

## Где AI действительно полезен

### Генерация дополнительных вопросов

На основе уже заполненных данных AI создает новые вопросы.

Например:

Исходные поля:

- Автор — Ван Гог
- Стиль — Постимпрессионизм
- Год — 1889

Дополнительные вопросы:

- К какому направлению относится работа?
- Какие художники работали в тот же период?
- Какие картины похожи по стилю?

### Анализ ошибок

После экзамена AI анализирует результаты.

Например:

> Вы часто путаете импрессионизм и постимпрессионизм.

или

> Большинство ошибок связано с французскими художниками XIX века.

### Персональные подборки

На основании истории обучения AI предлагает объекты для повторения.

### Генерация экзаменов

Пользователь может задать параметры:

- количество вопросов;
- сложность;
- темы;
- типы вопросов.

AI формирует экзамен автоматически.

### Учебный помощник

После ответа пользователь может задавать вопросы по объекту.

Например:

- Чем отличаются Моне и Мане?
- Почему эта картина относится к импрессионизму?
- Какие произведения похожи на эту работу?

AI отвечает, используя данные набора и внешние знания.

# 11. Возможности монетизации

Проект изначально проектируется с учетом возможности расширения коммерческой модели.

Возможные варианты:

- ограничение количества приватных наборов;
- увеличение лимита объектов в наборе;
- расширенная аналитика;
- командные рабочие пространства;
- совместное редактирование наборов;
- брендированные страницы экзаменов;
- AI-функции по подписке.

# 12. Долгосрочное развитие

В дальнейшем платформа может превратиться не просто в сервис создания экзаменов, а в экосистему визуального обучения.

Возможные направления развития:

- каталог публичных наборов с поиском и рейтингами;
- подписка на авторов и образовательные организации;
- совместное создание наборов;
- маркетплейс платных учебных наборов;
- мобильное приложение;
- API для интеграции с LMS (например, Moodle);
- импорт и экспорт наборов между пользователями.
